The short version

  • Traffic falls 2.6× / 4.5× (posit16 / posit8) on reuse-heavy kernels — and inverts on single-pass kernels, which is the honest caveat.
  • 7.5× faster cycles on gemm 128³ once the narrow panel fits L1 (dataset v2, upgraded core timing model); with tiling + packing equalized, narrowing holds cycle parity at 1.9–3.5× less traffic.
  • posit32 + quire ≥ f32 accuracy at the same 4 bytes — exact on cosine_sim, 8.9× better than f32 on gemm 256³.
  • posit8 is a bandwidth probe, not a number format — its error saturates (142× on conv2d), which is exactly what the cost model is built to refuse. int8 fails the same kernel the same way (31×).
  • Kahan dies where naive f32 dies on the ill-conditioned sweep (~1e8) and returns 0.0 on the orthogonal probe — compensation fixes drift, not cancellation; the quire is exact to 1e18.
